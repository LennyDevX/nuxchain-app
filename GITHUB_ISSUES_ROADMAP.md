## 🏷️ Label Definitions

- `critical`: Blocking or essential for platform operation
- `feature`: New functionality
- `enhancement`: Improvement to existing functionality
- `bug`: Something isn't working
- `security`: Security-related issue
- `smart-contract`: Smart contract development
- `ui`: User interface changes
- `documentation`: Documentation improvements
- `phase-1`, `phase-2`, `phase-3`: Roadmap phase
- `mobile`: Mobile-specific
- `infrastructure`: Backend/infrastructure work
- `testing`: Testing and QA
- `good-first-issue`: Good for newcomers
- `help-wanted`: Extra attention needed

```markdown

[Clear description of the feature/bug]

- [ ] Criterion 1
- [ ] Criterion 2
- [ ] Criterion 3

## Technical Requirements
- Requirement 1
- Requirement 2

## Dependencies
- Issue #X must be completed first
- Requires Y library/service

## Estimated Effort
[Small / Medium / Large / X-Large]

## Additional Context
[Any additional information, mockups, or references]
```

## 🤝 Contributing

Developers interested in working on these issues should:

1. **Check the issue status** - Ensure it's not already assigned
2. **Review dependencies** - Make sure prerequisite issues are completed
3. **Read acceptance criteria** - Understand what "done" looks like
4. **Ask questions** - Comment on the issue if anything is unclear
5. **Create a branch** - Use naming convention: `feature/issue-number-brief-description`
6. **Submit PR** - Reference the issue number in your pull request
